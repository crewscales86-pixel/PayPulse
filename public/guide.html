SCENE 1 · WHAT IT DOES
What is PayPulse?
PayPulse automatically bills your clients based on triggers — when a lead books a call, when an appointment happens, when a campaign hits a milestone. You stop sending invoices. They get charged automatically.
⚡
Take a charge
Fire a one-off charge against any client's saved card with one click — no logging into Whop/Stripe.
🤖
Auto-charge on triggers
When a GHL appointment is booked, PayPulse can charge the client automatically.
📊
Track everything
CPL, CPBC, conversions, revenue per client — all in one dashboard.
🎯
Build quiz funnels
Drop-in qualification funnels with adaptive logic and Meta Pixel integration.
💡
You'll need: a payment processor (Whop, Stripe, or Fanbasis), about 15 minutes, and admin access to that processor.

SCENE 3 · WHOP SETUP
Get your Whop API Key
This key lets PayPulse talk to Whop on your behalf. Generated once, kept secret.
📍 In Your Whop Dashboard
  1. Go to whop.com and log in
  2. Click your business name in the top-left corner — make sure you're inside the right business
  3. Click Settings (gear icon in the left sidebar, bottom)
  4. Click Developer
  5. Click the API Keys tab
  6. Click Generate API Key (or copy your existing one)
  7. Copy the key — it looks like whop_api_abc123...
⚠️
Keep this key secret. Anyone with it can charge your clients. If it leaks, regenerate it in the same place — old key stops working instantly.
📌
While you're in there, also grab your Company ID — it's in the URL bar (looks like biz_xxx) or under Settings → General.
SCENE 3 · STRIPE SETUP
Get your Stripe Secret Key
This is the master key that lets PayPulse create charges on your Stripe account.
📍 In Your Stripe Dashboard
  1. Go to dashboard.stripe.com and log in
  2. Make sure you're in LIVE mode — the toggle in the top-right should say "Viewing live data" (not Test mode)
  3. Click Developers in the left sidebar
  4. Click API Keys
  5. Find the Secret key row under "Standard keys"
  6. Click Reveal live key
  7. Copy the key — it starts with sk_live_...
🚫
About IP restrictions: Stripe may prompt you to set IP allowlist restrictions. Leave this blank and click Save without adding anything. PayPulse runs on dynamic infrastructure and IP restrictions will break it.
⚠️
If your key starts with sk_test_, you're in Test mode. Switch to Live mode (top right) before copying.

SCENE 5 · WEBHOOKS (CRITICAL)
Set up your webhook
This is the most important step. Without the webhook, PayPulse doesn't know when your clients pay or add new cards. It's how PayPulse "listens" to your payment processor.
Client pays
via your checkout
→
Whop
processes the charge
→
Webhook fires
sends event to FC
→
PayPulse
saves the card
📍 Step 1 — Get Your Webhook URL from PayPulse
  1. In PayPulse, click Settings (top-right)
  2. Scroll to the Webhooks section
  3. Find the URL for your processor — it looks like https://PayPulse.app/webhook/whop/abc123...
  4. Click Copy
📍 Step 2 — Add the Webhook in Whop
  1. Back in Whop → Settings → Developer
  2. Click the Webhooks tab
  3. Click Create Webhook (or Add Endpoint)
  4. Paste your PayPulse URL in the URL field
  5. Select these events: payment.succeeded and membership.activated
  6. Click Save
📍 Step 2 — Add the Webhook in Stripe
  1. Back in Stripe → Developers → Webhooks
  2. Click + Add Endpoint
  3. Paste your PayPulse URL in the Endpoint URL field
  4. Click + Select events
  5. Add these events: payment_intent.succeeded and payment_intent.payment_failed
  6. Click Add events, then Add endpoint
✅
Test it: after saving the webhook, click "Send test event" in your processor. Within 30 seconds you should see a test customer appear in PayPulse → Customers tab. That confirms it's wired correctly.

📍 How It Works
  1. Click any customer card → Charge
  2. Enter the amount (in their currency)
  3. Add an optional note ("Setup fee", "Make-good for missed appt", etc.)
  4. Click Fire Charge
  5. The charge fires against their saved card. Card is never re-entered.
⚡
What about automatic charges? You can also set up triggers that fire automatically — e.g. when a GHL appointment is marked "Showed", PayPulse bills the client. That's set up in the customer's Edit panel under "Trigger Charge Amount".

Adaptive Funnel Coming Soon
Auto-adjusts based on the assigned client's performance
Every 72 hours, PayPulse compares this funnel's CPL/CPBC vs. the customer's target. If things are bad, it auto-reduces questions to lower friction. If CPL is fine but bookings are expensive, it boosts the intent value sent to Meta — telling Meta to optimize harder for quality.
RULE 1: CPL > 30% over target AND CPBC > 20% over → reduce questions (light or heavy depending on severity)
RULE 2: CPL ±20% but CPBC > 30% over → boost intent value to Meta (1.0–2.0x multiplier)

SCENE 12 · TROUBLESHOOTING
Common issues & fixes
Click any question to expand the answer. These are the most-reported issues from mentees in the past few months.
"No payment method on file" when trying to charge
For Whop customers: click the 🔄 Pull from Whop button next to the Payment Method ID in the charge modal — fetches the latest card directly from Whop.

For Stripe/Fanbasis: the customer needs to complete a payment in the processor first. Once they pay, the webhook saves the card automatically.

You can also click Sync Customers to backfill cards for all clients at once.
Customer paid but isn't showing in PayPulse
Your webhook isn't configured correctly. Go to your processor's webhook dashboard → Event deliveries — look for failed (red) events. If you see 500 errors, click Resend on the failed event.

You can also manually pull all missing customers with Sync Customers.
Stripe webhook returning 500 errors
Usually transient — Railway may have restarted. Wait 60 seconds, then click Resend on the failed event in Stripe Dashboard → Developers → Webhooks → Event deliveries.

If errors persist:
• Verify your Stripe Secret Key starts with sk_live_ (not sk_test_)
• Verify the webhook URL matches exactly what's in PayPulse → Settings
Whop API returning 401 Unauthorized
Your Whop API key got rotated or is incorrect. Generate a new one in Whop → Settings → Developer → API Keys, then paste it into PayPulse → Settings.

Important: the key must be created at the sub-account (business) level, NOT the agency level.
"Invalid Private Integration token" (GHL Calendar integration)
Your GHL Private Integration token must be created INSIDE the sub-account that owns the calendar — not at agency level.

Switch into the sub-account first, then: Settings → Private Integrations → Create New Integration Token.

Required scopes: contacts.write, calendars.write, calendars/events.write, calendars.readonly.
Login session keeps expiring / "Invalid token"
Sign out and sign back in. The latest PayPulse auto-refreshes session tokens, but you need a fresh login to grab the refresh token first.

If it keeps happening, clear localStorage for the PayPulse domain in your browser dev tools, then log in fresh.
AI question generation returns nothing
PayPulse needs an ANTHROPIC_API_KEY environment variable set in Railway. If it's missing, the funnel builder shows fallback default questions instead.

Get a key from console.anthropic.com/settings/keys and add it in Railway → Variables.
Funnel disappeared from the dashboard
Funnels have an active flag. If it got toggled to false, the funnel still exists but is hidden. Re-activate it in the Funnels tab by clicking the Activate button on the card.
