SCENE 2 · WHAT IT DOES
What is PayPulse?
PayPulse is a pay-per-appointment billing system for SMMAs. You add clients, set their rate, and charge them when appointments happen. Charges are automatic via GHL webhooks or manual with one click.
📋
Kanban Board
Drag clients between New, Active, At Risk, and Paused columns. Visual pipeline at a glance.
⚡
Auto-charge
GHL webhook triggers charge the client's card when an appointment is booked or marked Showed.
📊
Metrics Dashboard
30-day revenue chart, top customers, trigger volume, failure rate — all in one view with interactive charts.
💡
You'll need: a Stripe account (recommended) or Whop account, and about 15 minutes to get everything connected.
SCENE 3 · CONNECT PAYMENTS
Connect your payment processor
Go to Settings in the left nav. Choose Stripe or Whop as your processor. Paste in your API keys — your keys stay encrypted in the database.
◆
Stripe (recommended)
Go to Stripe Dashboard → Developers → API Keys. Copy your Secret Key (sk_live_...) and Publishable Key (pk_live_...). Paste both into PayPulse Settings.
⚡
Whop
Go to Whop Dashboard → Settings → API Keys. Copy your API Key and Company ID. Paste into PayPulse Settings.
⚠️
Important: Use live keys (sk_live_...), not test keys. Test mode won't actually charge cards.
SCENE 4 · AUTO-BILLING
Connect your GHL webhook
When an appointment happens in GHL, PayPulse can auto-charge the client. This is the core of automated billing.
  • Go to Settings → Webhook URLs — copy your GHL Trigger URL
  • In GHL: Settings → Webhooks → Add Webhook — paste the URL
  • Select trigger: Appointment Created (or Appointment Showed for appointment tracking mode)
  • Save — test by booking a test appointment in GHL
    • POST /webhook/ghl/{secret}/ → auto-charges matched client
    ⚡
    Per-client routing: Each client can have a unique webhook URL. In the client edit modal, add their GHL Location ID. The URL becomes /webhook/ghl/{secret}/{locationId} — only charges that specific client.

    SCENE 9 · LEAD GEN
    Build qualification funnels
    Create AI-powered qualification funnels for specific niches. Each funnel asks questions, collects leads, and can forward them to GHL.
  • Go to Funnels → + New Funnel
  • Enter a Name and Niche/Description (e.g., "Roofing in Atlanta")
  • Click AI Generate — 4 qualification questions are auto-generated (requires ANTHROPIC_API_KEY)
  • Edit questions, set a headline, success message, and brand color
  • Optionally add a GHL Calendar ID to let leads book directly from the funnel
  • Toggle Active — your funnel is live at paypulse.app/f/{slug}
    • 🎯
    Leads submitted through the funnel are stored in PayPulse. You can view them from the Funnels page → click the funnel → Leads tab.
    SCENE 10 · CHARGE FAILURES
    Handle failed charges
    When a charge fails, PayPulse automatically moves the client to "At Risk" and sends you a notification. You can also set up an SMS alert via GHL webhook.
  • Go to Settings → Failed Charge Alerts
  • Create a webhook in GHL (Settings → Webhooks → Add Webhook)
  • Paste the GHL webhook URL into PayPulse Settings
  • Now when a charge fails, PayPulse sends the client name, company, amount, and failure reason to GHL
  • Create a workflow in GHL that sends an SMS to your team: "{client} - ${amount} charge failed"
    • POST /your-ghl-webhook → { "event": "charge.failed", "customer_name": "...", "customer_company": "...", "amount": 147, "failure_reason": "card_declined" }
    SCENE 11 · TROUBLESHOOTING
    Common issues & fixes
    Quick fixes for the most common questions.
    ❌ "Charge failed" — card declined +
    Update the client's card. Go to Analytics → Card Info → Save New Card (opens Stripe Checkout). Or manually update in Stripe Dashboard. After saving, retry the charge from the client card.
    🔑 "Invalid Stripe key" +
    Make sure you're using a live Stripe key (starts with sk_live_). Test keys (sk_test_) won't process real charges. Go to Settings → paste your live Stripe Secret Key → Save.
    🌐 GHL webhook not triggering charges +
    Check that: (1) The GHL Trigger URL is copied correctly in Settings, (2) The client has their GHL Location ID set in Edit, (3) The client has a card on file, (4) The webhook is active in GHL. Test by booking a real appointment.
    🔒 "Account pending approval" +
    New accounts require admin approval before you can log in. Contact your admin or check back later. Once approved, you'll be able to log in with the credentials you signed up with.
    📊 Metrics shows $0 revenue +
    The Metrics page shows data from the last 30 days. If you just created charges, give it a moment or check that charges have created_at dates within the last 30 days. Admin can use 🌱 Seed Demo to generate test data.
